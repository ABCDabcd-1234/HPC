🌟 PARALLEL MINIMUM
🚀 KEY CONCEPT: REDUCTION
⚠️ Problem in Parallel Min

Suppose multiple threads do:

min_par = min(...)

simultaneously.

Race condition occurs.

✅ Solution:
OpenMP Reduction
🌟 Initialization
int min_par = INT_MAX;

Start with:

very large value
🚀 MOST IMPORTANT LINE
#pragma omp parallel for reduction(min:min_par)
🧠 What does reduction do?

Each thread gets:

private local copy

of:

min_par

Then OpenMP combines results automatically.

🧵 Example

Array:

5 2 9 1 7 3

4 threads:

Thread	Elements	Local Min
T1	5 2	2
T2	9 1	1
T3	7	7
T4	3	3

Final reduction:

min(2,1,7,3)=1
🌟 Why Reduction Important?

Without reduction:

threads overwrite shared variable

Incorrect results occur.

🌟 PARALLEL MAXIMUM
✅ Sequential Max
max_seq = max(max_seq, arr[i]);

Finds largest element.

🚀 Parallel Max
#pragma omp parallel for reduction(max:max_par)

Each thread computes local maximum.

OpenMP merges:

largest value
🧠 Example

Array:

3 9 2 11

Threads:

T1 → 9
T2 → 11

Final:

max(9,11)=11
🌟 SEQUENTIAL SUM
✅ Sum Variable

🔄 Accumulation

🌟 PARALLEL SUM
🚀 Reduction
#pragma omp parallel for reduction(+:sum_par)
🧠 What happens?

Each thread:

computes local partial sum

Then OpenMP:

adds all local sums
🧵 Example

Array:

1 2 3 4 5 6

Threads:

Thread	Elements	Local Sum
T1	1 2	3
T2	3 4	7
T3	5 6	11

Final:

3+7+11 = 21

🎯 Viva One-Liners
What is reduction?

Reduction combines thread-local results into one final value safely.

Why use reduction?

To avoid race conditions during parallel updates.

What is OpenMP?

OpenMP is an API for shared-memory parallel programming.

Why parallel faster?

Multiple CPU cores process data simultaneously.

Why long long used for sum?

Sum may exceed integer range.

What is race condition?

Multiple threads modifying shared data simultaneously causing incorrect results.
