🧠 What is OpenMP?
✅ OpenMP = Open Multi-Processing
It is an API used for parallel programming in C/C++.
👉 It allows multiple CPU threads to run simultaneously.
Instead of:
1 thread doing all work
We get:
many threads sharing the work

⚡ Why use OpenMP?
Normally:


CPU executes instructions sequentially


With OpenMP:


multiple CPU cores work together


improves performance


reduces execution time



🧵 Example
Suppose loop:
for(int i=0; i<8; i++)
Without OpenMP:
1 thread executes all 8 iterations
With OpenMP:
#pragma omp parallel for
Maybe:


Thread 1 → i = 0,1


Thread 2 → i = 2,3


Thread 3 → i = 4,5


Thread 4 → i = 6,7


👉 Work divided automatically

📚 What is BFS?
Breadth First Search
Traverses graph:


level by level


Example graph:
        0      /   \     1     2    / \     \   3   4     5
BFS order:
0 1 2 3 4 5
Uses:


Queue


Visits neighbors first



📚 What is DFS?
Depth First Search
Traverses:


depth-wise


DFS order:
0 1 3 4 2 5
Uses:


Recursion or Stack

✅ Function
void parallelBFS(int start)

vector<bool> visited(V, false);
Tracks visited nodes.
Initially:
false false false false false false

vector<int> current, next;

next

🚀 OPENMP PARALLELISM

✅ Parallel For
#pragma omp parallel for
This is the MOST IMPORTANT part.

What it does?
Suppose:
current = {1,2,3,4}
OpenMP distributes work:
ThreadNodeT11T22T33T44
All processed simultaneously.

✅ Node Processing
int node = current[i];
Each thread gets one node.

⚠️ Critical Section
#pragma omp criticalcout << node << " ";

What is critical?
Only ONE thread allowed at a time.
Without it:
threads print together → messy output
Example bad output:
012435
Critical makes printing safe.

🔍 Neighbor Traversal
for (int neighbour : adj[node])
Checks neighbors.
Example:
adj[1] = {0,3,4}

⚠️ Race Condition Problem
Two threads may try:
visited[4] = true
simultaneously.

✅ Solution
#pragma omp critical
Protects shared data:


visited


next



✅ Double Check
if (!visited[neighbour])
inside critical avoids duplicate insertion.

🔄 Level Update
current = next;next.clear();
Moves to next BFS level.

🧠 Example Execution
Initial:
current = {0}
Process:
0 → adds 1,2
Next:
current = {1,2}
Then:
1 → adds 3,42 → adds 5
Next:
current = {3,4,5}
Done.

🧠 DFS Flow

⚠️ Why DFS is NOT parallel here?
DFS depends on:


recursive order


stack dependency


Parallel DFS is complicated.
So:
DFS kept sequential
which is correct.
