🌟 BUBBLE SORT SECTION
📚 What is Bubble Sort?

Bubble sort repeatedly:

compares adjacent elements
swaps if incorrect

Largest element “bubbles” to end.

🧠 Example

Array:

5 3 1 4

Pass 1:

5 3 → swap
3 5 1 4

5 1 → swap
3 1 5 4

5 4 → swap
3 1 4 5
🌟 Sequential Bubble Sort
✅ Function
void sequentialBubbleSort(vector<int>& arr)

Sorts array normally.

✅ vector<int>& arr

Pass by reference.

Meaning:

original array modified directly

No copying.

✅ Size
int n = arr.size();

Gets array length.

🔄 Outer Loop
for (int i = 0; i < n - 1; i++)

Controls number of passes.

Bubble sort requires:

n-1 passes
🔄 Inner Loop
for (int j = 0; j < n - i - 1; j++)

Compares adjacent elements.

🔍 Comparison
if (arr[j] > arr[j + 1])

If left element larger:

swap needed
🔄 Swap
swap(arr[j], arr[j + 1]);

Exchanges positions.

⚠️ Complexity

Bubble sort:

O(n²)

Very slow for large data.

🌟 PARALLEL BUBBLE SORT
⚠️ Problem with Normal Parallel Bubble Sort

This is WRONG:

#pragma omp parallel for
for (j=0...)

because:

adjacent swaps interfere

Example:

j=0 swaps arr[0], arr[1]
j=1 swaps arr[1], arr[2]

Both use:

arr[1]

Race condition occurs.

✅ Solution:
Odd-Even Transposition Sort

This is parallel-safe bubble sort.

🌟 EVEN PHASE
for (int j = 0; j < n - 1; j += 2)

Compares:

(0,1)
(2,3)
(4,5)

These pairs are independent.

🧠 Why safe?

No overlapping indices.

Example:

(0,1)
(2,3)

No shared elements.

So:

parallel execution safe
🚀 OpenMP Parallelism
#pragma omp parallel for

OpenMP divides iterations across threads.

Example:

Thread	Work
T1	(0,1)
T2	(2,3)
T3	(4,5)
🌟 ODD PHASE
for (int j = 1; j < n - 1; j += 2)

Compares:

(1,2)
(3,4)
(5,6)

Again independent.

🧠 Complete Bubble Sort Flow

Pass:

Even phase
Odd phase

Repeated:

n times

Eventually sorted.

🌟 MERGE SORT SECTION
📚 What is Merge Sort?

Divide and conquer algorithm.

Steps:

Divide array
Recursively sort halves
Merge sorted halves
🧠 Example
8 3 5 1

Split:

8 3 | 5 1

Further:

8 | 3
5 | 1

Merge:

3 8
1 5

Final:

1 3 5 8
🌟 MERGE FUNCTION
✅ Function
mergeVectors(left, right)

Combines:

two sorted arrays

into one sorted array.

🔄 Two Pointers
int i = 0, j = 0;

Track positions.

🔍 Comparison
if (left[i] <= right[j])

Smaller element added first.

🔄 Push Element
result.push_back(...)

Adds element to merged array.

🌟 Sequential Merge Sort
✅ Base Case
if (arr.size() <= 1)

Single element already sorted.

✅ Midpoint
int mid = arr.size() / 2;

Splits array.

✅ Left Half
vector<int> left(...)
✅ Right Half
vector<int> right(...)
🔄 Recursive Calls
left = sequentialMergeSort(left);

Sorts left recursively.

🔄 Merge
return mergeVectors(left, right);

Combines sorted halves.

⚠️ Complexity

Merge sort:

O(n log n)

Much faster than bubble sort.

🌟 PARALLEL MERGE SORT
🚀 Key Idea

Left half and right half are independent.

Can run simultaneously.

✅ Threshold
if (arr.size() <= 1000)

Small arrays:

use sequential version
⚠️ Why?

Creating threads has overhead.

For tiny arrays:

parallel slower
🌟 PARALLEL SECTIONS
✅ MOST IMPORTANT PART
#pragma omp parallel sections

Creates parallel region.

🌟 SECTION 1
#pragma omp section
left = parallelMergeSort(left);

One thread sorts left half.

🌟 SECTION 2
#pragma omp section
right = parallelMergeSort(right);

Another thread sorts right half.

🧠 Example

Array:

8 3 5 1

Thread 1:

8 3

Thread 2:

5 1

Both simultaneously.

🎯 Viva One-Liners
What is OpenMP?

OpenMP is an API for shared-memory parallel programming.

Why odd-even transposition used?

Normal bubble sort causes race conditions during parallel swaps.

Why merge sort parallelizable?

Left and right halves are independent.

Why threshold used?

To avoid excessive thread creation overhead.

What is speedup?

Ratio of sequential execution time to parallel execution time.
